# Using HuggingFace Embeddings (Free — No API Key Required)

Follow this guide if you do not have an OpenAI API key and want to use a free local embedding model instead.

---

## Why This Change Is Needed

The file `src/core/vectore_store.py` currently uses OpenAI to convert text into vector embeddings when building and querying the RAG index. If you do not have an OpenAI API key, the `build_index.py` script and any RAG query in your agent will fail with an authentication error.

The fix is to swap the embedding model to `all-MiniLM-L6-v2`, which runs entirely on your local machine using the `sentence-transformers` library. This library is already installed — it is in `requirements.txt`. No new packages are needed.

---

## Step 1 — Edit `src/core/vectore_store.py`

Open the file. Find this block near the top (around line 8–27):

```python
from langchain_openai import OpenAIEmbeddings

# Initialize embeddings
embeddings = OpenAIEmbeddings(
    model="text-embedding-3-small",
    api_key=os.getenv("OPENAI_API_KEY")
)
```

**Remove those lines entirely and replace with:**

```python
from langchain_community.embeddings import HuggingFaceEmbeddings

# Initialize embeddings
embeddings = HuggingFaceEmbeddings(model_name="all-MiniLM-L6-v2")
```

That is the only code change required. The rest of the file stays exactly as it is.

---

## Step 2 — Rebuild the RAG Index

After saving the file, run:

```bash
python build_index.py
```

On the first run, the model (`all-MiniLM-L6-v2`, ~80 MB) downloads automatically from HuggingFace Hub. This requires an internet connection for the first run only. After that it is cached locally at `~/.cache/huggingface/` and works offline.

You will see output like:
```
INFO — Loaded 7 documents
INFO — Created 52 chunks
INFO — Vector store created at data/vector_store
```

---

## Important — Do Not Mix Embedding Models

The vector store saves embeddings as numerical vectors. Vectors generated by OpenAI and vectors generated by HuggingFace are **not compatible** — they use different dimensions and scales.

If you switch embedding models, you must always rebuild the index before running your agent. Otherwise queries will return garbage results or errors.

| Scenario | What to do |
|----------|------------|
| You previously built the index with OpenAI and now want HuggingFace | Make the code change above, then run `python build_index.py` again |
| You built with HuggingFace and want to switch back to OpenAI | Revert the code change, add `OPENAI_API_KEY` to `.env`, run `python build_index.py` again |
| You just added new documents to `data/knowledge_base/` | Run `python build_index.py` again — no code change needed |

---

## Verifying It Works

After rebuilding, test that the vector store is queryable by running a quick search from the Python REPL:

```python
from src.core.vectore_store import search_vector_store

results = search_vector_store("minimum test cases per story", top_k=2)
for doc, score in results:
    print(score, doc.page_content[:100])
```

If you get results back with text from your knowledge base documents, everything is working correctly.
